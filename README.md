# Projekt. Fibonacci-ALGOSCHULE

[Русский](./README.ru.md/)

## Ziel des Projekts

Das einseitige React-Anwendung lässt visuel kennenlernen, wie unten genannten Algorythmen funktionieren:

- **die Wende der Zeile**;
- **die Kalkulation die Fibonacci-Folge**. Die Anzahl der Zahlen in der Folge wird von der Benutzter von 1 bis 19 bestimmt;
- **das Sortieren des Arrays**. Zwei Sortiermethoden sind implementiert: „Bubble“ und „Auswahl“. Die Sortierung ist sowohl aufsteigend als auch absteigend möglich. Der Benutzer bestimmt die Sortiermethode und -richtung;
- **Stack**. Das Hinzufügen und Entfernen von Elementen zum Stack ist implementiert;
- **Queue**. Der Benutzer kann Elemente zur Queue hinzufügen und daraus entfernen;
- **Verkettete Liste**. Mithilfe von Steuerelementen können Elemente zu einer verketteten Liste hinzugefügt und daraus entfernt werden.

## Lernziele des Projekts

Dieses Projekt dient der Weiterbildung und wurde im Rahmen eines Kurses bei Yandex.Practicum in zwei Phasen abgeschlossen.

Ziel der ersten Phase war die Visualisierung der behandelten Algorithmen mithilfe der von Yandex.Practicum bereitgestellten Vorlage. Diese Arbeit umfasste nicht nur das Schreiben des Codes zur Implementierung des Algorithmus, sondern auch die Erstellung einer Animation – einer schrittweisen Demonstration der Funktionsweise des Algorithmus. Daher musste der Student Animations- und Implementierungsplanungsfähigkeiten beherrschen, ein tieferes Verständnis der Reihenfolge von Array-Sortieralgorithmen erlangen und die Implementierung von Datenstrukturen wie Stacks, Queues und verketteten Listen erlernen.

In der zweiten Projektphase mussten die Studenten Unit- und End-to-End-Tests gemäß der Aufgabenstellung schreiben und so ihre anfänglichen Kenntnisse im Testen von Code festigen.

## Verfügbare Skripte

Im Projektordner können Sie die folgenden Befehle ausführen:

### `npm run start`

Das Projekt wird im Entwicklungsmodus gestartet und im Browser unter http://localhost:3000 geöffnet. Die Seite wird nach dem Speichern von Codeänderungen neu geladen.

### `npm run build`

Ein Build der Arbeitsversion des Projekts wird erstellt. Die Build-Ergebnisse werden im Ordner `build` abgelegt.

### `npm run test`

Unit-Tests wird gestartet.

### `npm run cypress:open`

Die Testumgebung für End-to-End-Tests wird gestartet.

**Wichtig!!!** Sie müssen das Projekt zuerst mit dem Befehl `npm run start` im Entwicklungsmodus starten.

### `npm run deploy`

Das Projekt wird auf GitHub Pages gehostet.
